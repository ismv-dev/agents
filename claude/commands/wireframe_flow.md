# wireframe_flow

**Propósito:** Trazar la arquitectura de la información y la experiencia de navegación que seguirá el usuario final.

## Instrucciones
1. Mapea el flujo de pantallas que recorre un usuario para cumplir con las épicas del `PRD_INITIAL.md`.
2. Detalla los disparadores de eventos (clicks, envíos de formulario) y los cambios de estado visuales que ocurren en la navegación.
3. Especifica los caminos alternativos (e.g., flujo de error de inicio de sesión).

## Restricciones
- Evita descripciones narrativas largas; utiliza representación gráfica en formato de código plano.

## Salidas Verificables (Artifacts)
- Archivo `/ui/USER_FLOW.md` que contenga un diagrama de flujo estructurado utilizando la sintaxis nativa de **Mermaid.js**.
