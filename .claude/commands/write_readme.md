# write_readme

**Propósito:** Estructurar el punto de entrada principal del conocimiento técnico del proyecto para ingenieros y colaboradores.

## Instrucciones
1. Recopila la información técnica de los artefactos de todos los agentes (DevOps, Backend, Frontend).
2. Redacta secciones imperativas: Prerrequisitos del sistema, Guía de instalación paso a paso, Comandos para levantar el entorno de desarrollo local, Arquitectura del directorio y Guía de contribución limpia.
3. Diseña una documentación altamente legible, visualmente organizada e intuitiva.

## Restricciones
- Prohibido dejar instrucciones desactualizadas o rutas de instalación que requieran suposiciones por parte del desarrollador.

## Salidas Verificables (Artifacts)
- Archivo `README.md` maestro en la raíz del proyecto, perfectamente formateado y libre de enlaces rotos.
